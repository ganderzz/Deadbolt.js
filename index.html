Deadbolt.js

Deadbolt is a lightweight form validator that uses data attributes to define the rules.

API

Validators

Validators are the rules given to an input. These can be defined like:
<input type="text" data-valid="digit,minLength=5,not=1" /> Multiple validators can be separated by commas. When Addons are created, they are applied to an input in the same way.

List of predefined Validators:
  • email: checks if input matches an email format (ex. email@mail.com)
  • minLength: checks if the input has a minimum character length (Defined with an argument, minLength=5)
  • not: checks in the input does not match the defined argument (not=welcome)
  • digit: checks if the input is either a digit or symbol

Hello World?


var cache = $(".myForm").deadbolt();
            
if(cache.isValid())
  console.log("The form is valid!");

Deadbolt caches validators defined by data attributes. These validation rules are cached, then checked against the input once isValid() is called. True or false are returned by isValid() depending if all inputs are valid.

Addons


var cache = $(".myForm").deadbolt({
                    endsWithIt: function(inputVal) {
                        return new RegExp(".+(it)", "i");
                    }
                });
            
if(cache.isValid())
  console.log("The form is valid!");
else
  console.error("myForm is NOT valid!");

Addons are parameters passed to deadbolt that provide the ability to add additional rules. These rules can then be used by data-valid. For now, returned values must be new RegExp statements.
